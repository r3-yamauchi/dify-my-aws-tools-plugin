# プライバシーポリシー

本ドキュメントは、Dify 用プラグイン **my_aws_tools** がデータをどのように取り扱うかを説明します。プラグインは AWS の各種サービス（Bedrock、SageMaker、Lambda、Transcribe、OpenSearch、S3、DynamoDB など）との連携に特化しており、ユーザーがツールを実行する際に必要なデータのみを扱います。解析目的のテレメトリは収集しません。

## データ収集
- **ユーザー入力**: テキストプロンプト、音声/動画 URL、翻訳パラメータ、Lambda へのリクエストなど、ユーザーが明示的に渡した値だけを AWS へ転送します。
- **設定メタデータ**: 任意の AWS 資格情報（アクセスキー、シークレットキー、リージョン）はプロバイダー設定またはツール単位で指定できます。これらはプラグインの実行プロセス内に留まり、AWS SDK クライアントの認証にのみ使用されます。
- **生成結果**: Bedrock 検索結果、Transcribe の書き起こし、SageMaker 推論結果、ブラウザーセッション情報など、AWS から取得したレスポンスはそのまま Dify へ返却し、プラグイン内部で保存しません。
- 個人情報はユーザー入力に含まれる場合のみ扱います。

## データ利用
- 収集したデータは、指定された AWS ツールを実行するためだけに使用されます（例: Transcribe を実行する、Bedrock の Knowledge Base を検索する、Nova で画像/映像を生成する 等）。
- AWS からの出力は結果整形を除き二次利用しません。他目的のために保存・再処理・学習に利用されることはありません。

## データ保存
- プラグイン自身はユーザー入力や出力を恒久的に保存しません。
- GIF からフレームを抽出する等の処理で一時ファイルを作成する場合がありますが、処理完了後に削除します。
- S3 や DynamoDB などにデータを保存したい場合は、ユーザーが該当ツールで明示的に指示したときのみ実行され、そのデータはユーザーの AWS アカウント側に保存されます。

## 第三者サービス
- 通信先は AWS サービス（Bedrock、SageMaker、Lambda、Transcribe、OpenSearch、S3、DynamoDB など）および Bedrock AgentCore Browser + Playwright のみです。
- AgentCore Browser ツールでは、セッションの WebSocket URL やヘッダーを AWS Systems Manager Parameter Store に保存しますが、これはユーザーの AWS アカウント内で管理されます。
- それ以外の第三者サービスへデータを送信することはありません。

## セキュリティ
- すべての AWS 連携は HTTPS 経由で行い、資格情報は boto3 クライアント内にのみロードされます。プロバイダーで指定した資格情報はディスクへ永続化されません。
- Parameter Store に保存されるブラウザーセッション情報は、ユーザーの IAM ポリシーに従って保護されます。
- Playwright セッションやメモリキャッシュはプロセス終了時またはセッション終了時に破棄されます。
- 一時ファイル（音声ダウンロードや GIF 抽出など）は厳格な権限で保存し、処理完了後削除します。
- AWS リソース（IAM、S3 バケット、DynamoDB テーブル等）のセキュリティ設定は利用者側の責任で管理してください。プラグインは与えられた権限内でのみ動作します。

ご質問やプライバシーに関する懸念がある場合は、リポジトリの issue からお問い合わせください。
